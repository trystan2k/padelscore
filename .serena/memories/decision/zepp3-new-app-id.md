## What
The Zepp OS API Level 3+ migration will use a new appId rather than continuing the existing marketplace identity.

## Why
The user explicitly confirmed that the migrated app should be published as a new app/listing identity.

## Where
Affects `app.json`, release/distribution strategy, store submission, and migration rollout planning.

## Learned
This removes the need to preserve listing continuity with the Zepp OS 1.0 app, but it introduces naming/versioning/release communication work for the new app package.
